Design Section 1: Data & Security

Extending the database and establishing the auth flow

Database Schema Changes
-- Add roles and passwords
ALTER TABLE users 
ADD COLUMN password_hash TEXT,
ADD COLUMN role TEXT DEFAULT 'user',
ADD COLUMN created_at TIMESTAMP DEFAULT NOW();

-- Create sessions table for secure persistence
CREATE TABLE web_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    expires_at TIMESTAMP NOT NULL
);
The Auth Flow
  1. POST /login: Validates email/hash
  2. Set-Cookie: HttpOnly, Secure session ID
  3. Middleware: SessionMiddleware loads user/role into context
  4. RBAC: RequireRole('admin') checks context before tool/page access

Does this data and security foundation look correct?

I will use golang.org/x/crypto/bcrypt for secure password hashing.
