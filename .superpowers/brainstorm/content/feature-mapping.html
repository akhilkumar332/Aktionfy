Feature Porting Strategy

Review the proposed features to port from the reference projects into our Go backend.

1. Human-in-the-Loop & Approvals

From Jules-MCP
  • Feature: `require_plan_approval` workflow for sensitive scheduled tasks.
  • Application: Tasks that execute shell commands or modify files wait for explicit user approval via the dashboard before the server executes them.

2. Secure Persistence & Secrets

From Jules-MCP
  • Feature: AES-256-GCM encryption for stored schedules and credentials.
  • Application: If users provide API keys (e.g., GitHub token) for a scheduled task, they are encrypted in PostgreSQL and only decrypted at execution time in memory.

3. Rich Markdown & JSON Returns

From schedule-task-mcp
  • Feature: Returning both human-readable Markdown summaries and raw JSON for automation.
  • Application: When Gemini lists or creates tasks, the MCP server returns a beautifully formatted markdown table for the CLI, alongside JSON data.

4. Proactive AI Prompting

From Google Scheduled Actions
  • Feature: Natural language recurring prompts (e.g., "Every Friday at 5 PM, create a grocery list").
  • Application: Allowing users to schedule purely conversational/research prompts that automatically run and email the user or push to the dashboard.
